# Возможное развитие: авторизация и история

Статус: аккаунты и серверное хранение находятся вне текущего MVP. В демоверсии реализована локальная история восьми запросов в текущем браузере.

В текущей демоверсии используется один общий код доступа перед интерфейсом поиска. Код задаётся в окружении Yandex Cloud Function, проверяется отдельным запросом при входе и повторно проверяется сервером при каждом поиске. Он не является полноценной пользовательской авторизацией и предназначен только для ограничения публичного доступа.

Если FoodSup станет рабочим продуктом компании, отдельным этапом потребуются:

- учётные записи сотрудников и роли;
- безопасные сессии и восстановление доступа;
- личная синхронизируемая история поисковых запросов;
- серверное хранение заметок и списков поставщиков;
- журнал административных действий;
- управление пользователями и отзыв доступа;
- политика хранения и удаления данных.

Выбор конкретного сервиса авторизации и базы данных откладывается до появления реальных требований по числу пользователей, инфраструктуре компании, резервному копированию и интеграции с корпоративной системой доступа.

Чтобы будущая доработка не потребовала переписывать поиск, пользовательские данные следует подключать через небольшие изолированные модули `authService` и `historyRepository`. Текущая логика веб-поиска, формирования карточек и расчёта соответствия от них не зависит.
